Projects

Financial Calculator

Small terminal app that I built with Python that can make financial calculations. Specifically it can calculate monthly loan payments and total interest paid over the life of the loan. It can also calculate dividend payments for interest bearing deposit accounts like CDs and Money Markets. Also made a GUI wrapper for it with the Tkinter library.

The project uses class objects in Python and uses class inheritance via a parent/child class relationship. The parent class acts as a sort of traffic director and controls the flow of the rest of the program based on user inputs. It prompts the user to pick a loan or savings project, then summons the next appropriate class object based on the user's choice. It asks for the user to input the interest rate when calculating loans and uses JSON objects to store tiered dividend rates for money market accounts where the dividend rate is determined by the average daily balance of the account.


The terminal version of the app uses some simple ASCII art to make the program more visually appealing. The GUI wrapper was made with the customtkinter library, which is a modified version of the tkinter library and offers a more modern look and feel.

Terminal RPG Game

This one is still in progress but has a significant amount of logic already in place. It is an RPG game that is played in a terminal and mimics 1990's style text based choose your own adventure games. Currently has logic in place for player characters, npcs, enemies and bosses. It also has party management logic, combat logic, control flow logic, and a logic tree to control the story. Currently written in Python but may consider rewriting later down the road to make it playable on the web.

This one is a self driven project and I assigned it to myself to give myself a more significant challenge than just writing something that can be done with a couple hundred lines of code, as was often the case with portfolio projects I was assigned. So I set out to design a game because that is definitely a project that requires many many lines of code, even in a simplified form like what I am currently working on.


In addition to the features I mentioned above, there is menu management (albeit plain text to fit the theme) and a save management system. The save files are JSON objects that capture the important details like player character attributes, location within the game, etc. At this point I want to get the story written before I work on any more features. Up to this point I have been testing the story tree with fragmented generic story elements just to make sure the logic works.

Board level repair of an Original Xbox

A coworker brought their Xbox to me and asked me if I could fix it. That one day it just wouldn't boot. I took it home and accepted the challenge. I've determined from the diagnostic lights that it is an overheat issue. But the machine hasn't started in a long time. I opened it, cleaned a little corrosion I found, and dusted it. With some help from Reddit I have determined the chip at U6F1 is likely bad/dead and needs to be replaced as that chip controls overheat signals and was near the previously mentioned corrosion.

I have received the microchip I was waiting for. So I am going to replace the offending heat sensor chip and see if that resolves the issue. If it doesn't my plan is to troubleshoot the PSU next.
